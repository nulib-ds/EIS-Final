<decisions_made>

- **Data source**: Used `eis-index.json` from the pipeline output (181 works) as a static asset rather than fetching individual IIIF manifests at runtime (too slow) or relying on Canopy's search index (format unknown/embedded in JS bundle)
- **State matching**: The index has normalized `state` values (e.g., "California") that match the STATES array in EISFilterBar. Matched against index `state` field, not the raw manifest `Main Location` string
- **Federal/International skipped**: Works with `state: "Federal / International"` don't show a "Same State" section — it would be too broad/unhelpful
- **Slug derivation**: Work page URLs derived by extracting filename from `manifestUrl` (e.g., `environmental-impact-statement-proposed-regulation.json` → `/works/environmental-impact-statement-proposed-regulation.html`)
- **Separator**: Used `border-top: 3px solid #6A5885` on the `.canopy-work--related` container instead of an `<hr>` — matches the existing design language (same purple used on `.canopy-work--header` and `.canopy-work--people`)

</decisions_made>
